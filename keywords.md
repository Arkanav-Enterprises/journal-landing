# Journal Lock — Keyword Research Report

Date: 2026-07-11 · Market: US / English · Site: journallock.app (new domain, ~5 months old, minimal authority)
Data sources: Google Autocomplete API (real query data), live SERP inspection (difficulty), volumes are informed estimates — no paid SEO API connected.

## Executive summary

Journal Lock sits at the intersection of two markets: **app blockers / screen time** (big, competitive, dominated by Opal/Freedom/one sec at the head) and **journaling apps** (big, dominated by Day One/Apple Journal at the head). The head terms are unwinnable for a new domain — but the SERPs for mid- and long-tail queries in the doomscrolling/app-blocker space are held by *small competitor apps' own blog guides* (Spool, ScreenBuddy, ScrollGuard), which proves a small app site can rank there with keyword-targeted guide pages.

Strategy: (1) own the branded + exact-match-mechanic queries defensively — a **different app also named "Journal Lock"** (id 6758279529) and a clone ("Log or Lock") already compete for them; (2) win long-tail transactional queries where intent → download is direct; (3) build informational guides that funnel to the app for the big "how to stop doomscrolling" cluster.

### Deliberately skipped

- "journal with lock", "diary with lock and key" — autocomplete shows **physical product intent** (amazon, walmart, kmart, "near me"). Wrong audience.
- "screen time app", "journal app", "focus app" head terms — Apple's own features + Day One/Opal own these SERPs. Difficulty 70+, no play for a new domain beyond mentions in copy.
- "morning journal" — SERP contested by newspapers named Morning Journal.

## Intent → conversion logic

Autocomplete evidence of download intent (the "actually gets downloads" filter): queries containing *app/free/iphone/ios* dominate every cluster — e.g. "stop doomscrolling **app free**", "digital detox **app ios**", "locked journal **app free**", "phone addiction **app iphone**". Every guide targets the base query + these modifiers in H1/H2s and answers with the app as the mechanism, not an afterthought.

### Round 2 gap analysis (2026-07-11)

Ran fresh autocomplete research against the shipped roster to check for real gaps. Found two with genuine volume and zero cannibalization risk, added above as #11–12. Explicitly rejected:
- **"opal app alternative" / "forest app alternative"** — decent volume, but `apps-to-stop-scrolling` already names Opal and Forest 10+ times in an honest comparison; a dedicated page would cannibalize that guide's ranking equity, not add reach.
- **"screen time not working [on iphone]"** — real volume but wrong intent; people debugging an existing Screen Time bug aren't shopping for a new app.
- **"app blocker android"** — consistent volume across every cluster we checked, but Journal Lock is iOS-only. Not worth chasing unconvertible traffic.
